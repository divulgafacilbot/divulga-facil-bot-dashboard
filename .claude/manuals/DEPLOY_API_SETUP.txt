Issue: Build fails with "Cannot find module"
Solution:
  - Check package.json has all dependencies
  - Run npm install locally and commit package-lock.json
  - Verify import paths are correct

Issue: Database connection fails
Solution:
  - Verify DATABASE_URL is linked to PostgreSQL
  - Check PostgreSQL is in the same Railway project
  - Test connection locally with production DATABASE_URL

Issue: Environment variable not working
Solution:
  - Check variable name spelling (case-sensitive)
  - Verify variable is in Railway Variables tab
  - Redeploy after adding/changing variables

Issue: API returns 502 Bad Gateway
Solution:
  - Check logs for startup errors
  - Verify PORT is set correctly (Railway uses $PORT)
  - Ensure server is listening on 0.0.0.0, not localhost

Issue: CORS errors from frontend
Solution:
  - Verify CORS is configured in server.ts
  - Check APP_BASE_URL in environment variables
  - Ensure origin is set correctly in cors config

Issue: Migrations didn't run
Solution:
  - Manually run: railway run npm run migrate
  - Or execute SQL directly in Railway PostgreSQL Data tab
  - Check migration logs for errors

Issue: Webhook not receiving events
Solution:
  - Verify webhook URL is correct and accessible
  - Check HTTPS (not HTTP)
  - Test with curl or Postman
  - Review webhook logs in external service

Issue: High response times
Solution:
  - Check database query performance
  - Add database indexes if needed
  - Upgrade Railway plan for more resources
  - Implement caching (Redis)

Issue: Out of memory
Solution:
  - Check for memory leaks in code
  - Upgrade Railway plan
  - Optimize database queries
  - Implement pagination

1. Never commit .env files
   - Always in .gitignore
   - Use Railway Variables instead

2. Rotate secrets regularly
   - JWT_SECRET every 6-12 months
   - Webhook secrets after any breach
   - API keys quarterly

3. Use HTTPS only
   - Railway provides SSL automatically
   - Never disable SSL

4. Implement rate limiting
   - Prevent abuse
   - Protect against DDoS

5. Monitor access logs
   - Watch for suspicious activity
   - Set up alerts for anomalies

6. Keep dependencies updated
   - Run npm audit regularly
   - Update packages with security fixes

7. Use least privilege
   - Database users should have minimal permissions
   - API keys should have minimal scopes
